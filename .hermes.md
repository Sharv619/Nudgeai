# NudgeAI Hermes Project Rules

NudgeAI is a local-first personal productivity assistant. Treat it as a sensitive single-user system.

## Canonical Runtime

- Backend: `simple_api_server.py`
- Backend port: `8001`
- Frontend: `frontend/`
- Frontend dev server: `http://localhost:3000`
- Frontend API proxy: `/api` -> `http://localhost:8001`

## Safety Boundaries

- Do not require Hermes to run the core NudgeAI dashboard.
- Do not route core nudge CRUD through Hermes as the default product path.
- Do not directly edit `data/*.json` for product behavior unless the user explicitly asks.
- Prefer FastAPI endpoints over direct JSON mutation.
- Do not add autonomous messaging, booking, scheduling, or external actions without explicit review gates.
- Keep Google Calendar, Drive, location, health, and token artifacts local and out of commits.

## Never Commit

- `.env`
- `.env.*`
- `api/env.local`
- `token.json`
- `drive_token.json`
- `credentials.json`
- `client_secret*.json`
- `calendar_events.json`
- `drive_documents.json`
- `drive_documents_rag.json`
- `data_sync/*.json`
- real location, health, calendar, or Drive exports

## Required Verification Before Declaring Work Done

Run from the repo root unless noted:

```powershell
python -m py_compile simple_api_server.py mcp_api_bridge.py
python -m unittest discover -s tests -p "test_nudge*.py"
python scripts/privacy_check.py
python scripts/nudgeai_health_check.py
cd frontend
npm run build
```

## Hermes Integration Direction

Use Hermes in this order:

1. Local operations assistant.
2. Read-only nudge/context/source-status summaries.
3. Reviewed write actions through FastAPI after explicit approval.
4. Optional MCP wrapper only after the HTTP API and privacy model are stable.
5. Optional gateway delivery only after auth, audit, consent, and retention rules exist.
